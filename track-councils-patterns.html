Track D Councils & Multi-Agent

How to read this page

🔗
Six patterns, ordered by cost & complexity
Each pattern has a name, a token cost indicator, and a prompt template you can paste right now. Patterns 1, 2, and 6 are single-agent techniques that just look like multi-agent — they're the cheapest moves and where most ND professionals get the best ROI. Patterns 3, 4, and 5 are real multi-agent and earn their cost only when the structural conditions from the overview apply. If you've installed the panel-of-experts skill, it automates patterns 3 and 4.
1. Specialist Consultation ~1× 2. Devil's Advocate Solo ~1× 3. Pre-Mortem Council 3-5× 4. Parallel Research Dispatch N× 5. Review Board ~3× 6. Self-Debate ~1×
PATTERN 01 Specialist Consultation ~1× tokens
What it is. One domain question, one expert agent, one answer. No orchestration, no synthesis, no second opinion. The agent has a specialist system prompt and that's the entire architecture.
When to use
When the question is bounded and the domain is clear. "Does this CSS rule meet WCAG AA contrast?" goes to an accessibility-tester. "Is this clause problematic for a US freelancer?" goes to a legal-advisor. If you can name the one specialist who would answer it in real life, you can name the one agent.
ND value
Lowest cognitive overhead in the catalog. One question in, one answer out. Matches the ADHD pattern of resolving uncertainty and moving on — no waiting for a panel to synthesize, no parsing competing perspectives.
How to invoke 
Ask the [agent-name] agent: [your specific question with context]
Expected output
Direct answer in the specialist's voice, citing the specific rule or framework that governs the call. One pass, no back-and-forth.
PATTERN 02 Devil's Advocate Solo ~1× tokens · prompting technique
What it is. A single agent placed in explicit adversarial mode. This is not multi-agent — it's one agent with a different role. The cost is the same as any single-agent call.
When to use
Before any significant commitment: shipping a template, choosing an architecture, publishing a piece. Default-helpful agents unconsciously optimize for agreement. You have to explicitly ask for the opposite.
ND value
ADHD hyperfocus makes you vulnerable to overcommitment — once you're in love with an idea, you stop seeing its flaws. A scheduled adversarial pass is a forcing function for reconsideration that doesn't require willpower in the moment.
How to invoke 
You are a skeptical critic. Your job is finding every reason this
plan will fail. Be specific. Don't soften the critique to be kind.
For each failure mode, name the concrete mechanism that causes it.

Plan: [paste your plan]
Expected output
A numbered list of 5-10 specific failure modes, each with the concrete mechanism that causes it. If the output is vague hedging, your prompt isn't sharp enough.
PATTERN 03 Pre-Mortem Council 3-5× tokens · war room
What it is. Three to five domain experts run in parallel. Each independently identifies the most likely failure modes from their angle. An orchestrator (you, or the panel-of-experts skill) synthesizes consensus and conflicts. This is a real multi-agent move.
When to use
Major decisions where the cost of being wrong is high: product launches, architecture choices, pricing, public commitments. Reach for this when the answer needs to survive review by someone smarter than you tomorrow.
ND value
Converts the open-ended "what might go wrong?" (which produces ADHD analysis paralysis) into the bounded "we already failed — what went wrong?" frame. Concrete is always easier than abstract. The council also externalizes perspectives you'd otherwise try to hold in working memory all at once.
How to invoke 
Run a war room pre-mortem on [decision]. Assemble [3-5 experts].
Each expert independently:
  1. Names their 3 most likely failure modes from their domain
  2. Rates likelihood and severity (1-5)
  3. Proposes one mitigation per mode

Then synthesize: consensus failures, contested failures, and
the top 3 mitigations ordered by leverage.
Expected output
Per-expert failure lists, a synthesis section showing overlap and disagreement, and a prioritized mitigation plan. If the panel agrees on everything, your panel isn't diverse enough.
PATTERN 04 Parallel Research Dispatch N× tokens (N = subagents)
What it is. Multiple agents investigate independent questions simultaneously. This is the pattern Anthropic uses in production for their own research system — orchestrator dispatches, subagents research, orchestrator synthesizes.
When to use
Four or more independent questions that would otherwise take 4× longer in sequence. Independence is the test. If question B depends on the answer to A, the work is sequential and multi-agent buys zero speedup — only overhead.
ND value
Matches the natural mode of ADHD hyperfocus — "throw everything at the problem at once." Forcing yourself to decompose into truly independent subquestions before dispatch is the external scaffolding. The decomposition is the value, the parallelism is the bonus.
How to invoke 
# In Claude Code, dispatch multiple Task tool calls in a single
# message — they execute in parallel.

I need parallel research on [topic]. Dispatch these independent
investigations simultaneously:
  Subagent A: [specific question 1]
  Subagent B: [specific question 2]
  Subagent C: [specific question 3]
  Subagent D: [specific question 4]

When all return, synthesize into a single report with a
"contested findings" section.
Expected output
N parallel reports plus one synthesis. Time-to-answer roughly equals the slowest subagent, not the sum. If subagents end up calling each other, you decomposed wrong — the questions weren't actually independent.
PATTERN 05 Review Board ~3× tokens · sequential
What it is. A drafts, B critiques A's draft, C revises based on B's critique. Sequential, not parallel. This is the generator-verifier pattern with an explicit synthesis step.
When to use
Quality-critical deliverables that go out the door publicly: templates for sale, LinkedIn posts, course content, customer-facing copy. Anywhere the editing step is the part you reliably skip when running solo.
ND value
Externalizes the editing step. Editing is notoriously hard for ADHD brains — the hyperfocus that makes drafting feel good evaporates the moment the draft exists, and you lose objectivity in revision. A separate critique agent restores the missing perspective without requiring you to manufacture it.
How to invoke 
# Three sequential agent calls with explicit roles.

Call 1 (Drafter):
  Role: [domain] writer.
  Task: Draft [deliverable] for [audience] meeting [criteria].

Call 2 (Critic):
  Role: skeptical reviewer specializing in [angle].
  Task: Find 5-10 specific weaknesses in this draft. Cite passages.
  Input: [Call 1 output]

Call 3 (Reviser):
  Role: original drafter, now revising.
  Task: Address each critique. If you reject a critique, say why.
  Inputs: [Call 1 output] + [Call 2 output]
Expected output
Three artifacts: draft, critique, revised draft. Keep the critique — it's where the learning is, and you can reuse the same critic prompt next time.
PATTERN 06 Self-Debate ~1× tokens · prompting technique
What it is. One agent constructs the strongest case for both sides of a binary decision, then renders an explicit verdict naming the deciding factor. Single-agent throughout — the "debate" is structural, not architectural.
When to use
Binary decisions with genuine uncertainty. "Firebase or Neon?" "Free tier or paid at launch?" "Ship today or wait one more week?" Anywhere the decision is small enough to be one call but large enough that you've been turning it over for days.
ND value
Externalizes the weighing process that ADHD working memory struggles to hold — you can't compare two arguments fairly when you can only remember one at a time. Putting both on the page in their strongest form removes the recency-bias that makes whichever option you thought about last feel right.
How to invoke 
Decision: [X vs Y]

Step 1: Argue FOR option X specifically. Best case. Steel-man it.
        Cite the strongest evidence.
Step 2: Argue FOR option Y specifically. Best case. Steel-man it.
        Cite the strongest evidence.
Step 3: Render a verdict. Name the single factor that decided it.
        Note what would flip the verdict.
Expected output
Two steel-manned arguments and a verdict with one named deciding factor. If the verdict is "it depends," your prompt didn't force commitment — reframe the decision so commitment is possible.

The ordering matters

🧮
Master pattern 1 before reaching for pattern 3
The temptation, especially on a high-energy day, is to leap to the war room because designing a council feels more interesting than asking one good question. Resist. Most of what councils give you, a sharp specialist consultation already gives you at a fifth the cost. Patterns 1, 2, and 6 cost the same as a normal Claude call — they're free to practice and they handle the majority of real work. Patterns 3, 4, and 5 are the right move only when the question structurally needs them.
