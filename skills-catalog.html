🚫 Skills to Avoid 4 anti-recommendations
⚠️
Why this list exists
The Claude skill ecosystem has thousands of options. Star count is not a security signal. These skills are popular but have real risks — install them only if you fully understand the trade-offs.
asklokesh/loki-mode ↗ GitHub
Avoid. Requires --dangerously-skip-permissions. 37 autonomous agents with unrestricted file write, code execution, and network access. The same security pattern that caused 2 skills to be deleted from this catalog's source library — at 37× the blast radius.
prshahbs/healthcare-claude-skills ↗ GitHub
Avoid. Quality score 62/100 with documented data validation failures (invalid ICD codes accepted, extreme vital sign readings passing as valid). Healthcare misinformation from a low-quality source is a genuine harm category.
Any "curl … | bash" install pattern
Avoid. Cato Networks 2025 research confirmed Claude skills can be weaponized as malware delivery via this exact pattern — a "productivity" skill with a hidden payload in the install script. Always read the SKILL.md content directly on GitHub before cloning. Never pipe to bash blindly.
hesreallyhim/awesome-claude-code ↗ GitHub (42.8k★)
Avoid for now. 42,800 stars but currently under reconstruction — README explicitly says "Update in progress" and the Table of Contents is marked "TODO." Not usable as a skills discovery source right now. Check back later in 2026.

How to install any skill

📦
Three install patterns

1. Built-in / already on disk (the 🏠 Local tags above):
The skill file lives in ~/.claude/skills/[skill-name]/SKILL.md. Claude Code auto-loads it when trigger phrases match, or you can invoke explicitly: "Use the [skill-name] skill to..."

2. Plugin marketplace (the ✅ Official tags):
Inside Claude Code, run /plugin marketplace add anthropics/skills once, then /plugin install [skill-name] for each skill you want.

3. Manual git clone (the 🌐 Community tags):
git clone https://github.com/[author]/[repo] ~/.claude/skills/[skill-name] — restart Claude Code afterward. Read the SKILL.md file first to confirm what it does and what permissions it asks for.

🛡️
Safety check before installing any community skill
  1. Open the skill's SKILL.md on GitHub. Read it.
  2. Check the description for words like "unrestricted", "--dangerously", "sudo", "automatically execute". If you see any of these, stop and audit further.
  3. Check the install script (if any). Never run a script you haven't read.
  4. If unsure, run the security-reviewer skill on it first — it's literally designed for this.
One thing to remember: Install ONE new skill this week. Use it 3 times. Decide if it stays.
