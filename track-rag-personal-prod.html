When to graduate from Layer 1

Three concrete signals that a Layer 1 tool is no longer the right home for your knowledge:

📈
Trigger 1 — You've outgrown the file caps
You're routinely hitting "this Project is full" or "this notebook has too many sources." You're shuffling files in and out, or splitting one logical knowledge set across multiple Projects. The cap is making decisions for you instead of you.
🗂️
Trigger 2 — You already maintain a vault
Your notes live in Obsidian, Notion, or a folder of Markdown files, and you actively add to them. Re-uploading the same vault to a chat tool every week is duplication. A Layer 2 tool reads your existing vault in place.
🔒
Trigger 3 — Privacy is non-negotiable
Medical, legal, therapy, or financial content. Cloud opt-out toggles are policies, not technical barriers. Layer 2 local tools never send your documents over a network.

Layer 2 — Personal knowledge base

Obsidian + Smart Connections Fully local Free
Downloads: 786k+
Embedding: BGE-micro-v2 on-device
API key: Not required
Setup: 15–30 min

Obsidian is a free, local-first Markdown editor. Smart Connections is a community plugin that adds RAG on top — it embeds every note on your machine using a tiny open-source model (BGE-micro-v2), stores the index locally, and lets you query your entire vault through chat.

No internet call. No API key. No subscription. The v4 release in 2025 split the chat into its own plugin, so you can install only the embedding/search half if you don't want the chat UI.

You have 200+ notes, privacy matters, and Obsidian is already (or could be) your primary note system.
AnythingLLM + Ollama Network-isolatable Free
Setup: 60–90 min one-time
Formats: PDF, DOCX, MD, audio, more
Model: Local via Ollama
Ongoing cost: $0

An all-in-one desktop app that bundles a chat interface, RAG, document parsing, and audio transcription. Pair it with Ollama (which runs language models locally on your machine) and the whole stack stays on your hardware.

Version 1.12 added native tool-calling, Rust-based audio transcription, and AMD GPU support. RAG is built in from day one — unlike Open WebUI, where retrieval was added as an afterthought and shows it.

An ND-friendly setup walkthrough: Download AnythingLLM. Run the installer (one click). Open the app and let it auto-detect Ollama if you already have it, or follow the prompt to install Ollama next. Pick a model (start with llama3:8b — fast, capable, fits on most laptops). Create a workspace. Drag your folder of documents into it. Wait 5–10 minutes for indexing. Ask your first question. Done. You can stop here forever or tune later — both are valid.

Sensitive documents that cannot leave your machine, mixed formats including audio, and you want zero ongoing API cost.
Notion AI Q&A Cloud
Scope: Notion workspace only
Cross-doc reasoning: Limited
Plan: Notion AI add-on

Built-in RAG that lets you ask questions across your entire Notion workspace. Convenient if your knowledge already lives in Notion. The honest limitations:

  • Notion-only. It cannot reach Google Drive, your email, your downloaded PDFs, or anything outside Notion.
  • Chunk retrieval, not full-context. Cross-document reasoning is weaker than Claude Projects — it pulls matching snippets rather than holding multiple documents in working memory at once.
  • Cloud. Same training-opt-out story as the Layer 1 cloud tools.
Your knowledge already lives in Notion and you mostly want fast lookup, not deep cross-document synthesis.
Mem.ai Cloud
Price: $14.99/mo
Differentiator: Auto-organization
Privacy: Cloud — NOT for sensitive content

An AI-first notes app. Where Notion makes you file things, Mem files things for you — it auto-tags, auto-links, and surfaces related notes when you write. You capture, then query.

The caveat is non-negotiable: Mem is a cloud product. Anything sensitive does not belong here. For brainstorming, reading notes, and project journaling, it can be genuinely useful.

You capture a lot of small notes, hate manually organizing them, and your content is not privacy-sensitive.

Layer 3 — Production RAG (when it's actually warranted)

🏗️
The one question that justifies Layer 3

Are you building a product or feature for other people that has to retrieve from your data?

If yes — a customer support bot for your SaaS, a research tool for your team, an app feature shipped to users — Layer 3 is correct. You need version control, evaluation, observability, and the ability to scale.

If no — you're managing personal knowledge — close this page and go back to Layer 1 or 2. A homemade pipeline will cost more, fail more often, and consume the time you wanted to spend doing the thing.

For the people genuinely at Layer 3, here is the modern-2026 stack at a glance.

Vector databases — where embeddings live

Chroma

  • Free. In-memory or tiny Docker.
  • For prototypes and local dev.
  • Don't deploy to production at scale.

Qdrant

  • Best price-perf for production.
  • Self-hosted on a VPS: $30–50/mo.
  • Recommended default for small teams.

pgvector (Postgres)

  • Correct when Postgres is already in your stack.
  • One less moving part to operate.
  • Supabase free tier includes it.

Pinecone

  • Fully managed. Lowest-ops option.
  • Scales to thousands of dollars at high volume.
  • Pay for the "I don't want to think about it" premium.

Embedding models — how text becomes vectors

OpenAI text-embedding-3-small

  • $0.02 / million tokens.
  • Best quality-cost ratio default.
  • Start here unless you have a reason not to.

Voyage AI

  • $0.06–$0.12 / M tokens.
  • Leads MTEB retrieval benchmarks.
  • Worth the premium for retrieval-critical apps.

Cohere embed-v4

  • $0.01 / M — cheapest tier.
  • Unique search_document / search_query types.
  • Bulk indexing on a budget.

Local — BGE / nomic-embed-text

  • Free, runs via Ollama or on-device.
  • Slower indexing, full privacy.
  • Required for sensitive-content pipelines.

Frameworks — the glue code

LlamaIndex

  • Default recommendation. Data-intensive RAG.
  • 35% retrieval accuracy boost in 2025 release.
  • Lower per-query overhead (~1.6K tokens vs LangChain's 2.4K).

LangChain

  • Multi-step agent workflows beyond pure RAG.
  • Larger ecosystem, larger overhead.
  • Pick when you need agents + RAG, not RAG alone.

Haystack

  • Regulated industries — healthcare, legal, finance.
  • Auditable pipelines built in.
  • The compliance-friendly choice.

DSPy

  • Advanced. Treats RAG pipelines as optimizable programs.
  • Auto-tunes prompts and retrieval.
  • Adopt after you have a working baseline to optimize.

MCP servers for RAG — wire your RAG into Claude Code

🔌
Two notable RAG MCP servers

If you're already using Claude Code as your daily AI environment, an MCP server lets it query a local RAG index alongside your usual tools — no copy-pasting between apps.

  • knowledge-rag (lyonzin) — 12 MCP tools, 20 format parsers, hybrid keyword+semantic search with reranking. 100% local via ONNX runtime. The most feature-complete option.
  • mcp-local-rag (shinpr) — Smaller surface area, faster setup. Supports 5 parallel queries without performance degradation. Good fit when you want one MCP server doing one thing well.

Both install via standard MCP config. Both keep your index local. Pick the first for power, the second for simplicity.

Evaluation — does your RAG actually work?

📊
RAGAS — the production-grade evaluation

RAGAS (Retrieval-Augmented Generation Assessment) is the standard framework for measuring whether a RAG system is reliable. It scores four dimensions: faithfulness, answer relevance, context precision, context recall.

The one number to watch: faithfulness. This measures whether the AI's answer is grounded in retrieved documents versus inventing.

  • Faithfulness > 0.8 — production-ready for internal tools.
  • Faithfulness > 0.85 — acceptable for customer-facing features.
  • Below 0.7 — fix retrieval before shipping.

If you're not building a product, you don't need to run RAGAS. The 10-question spot-check from the previous page is the non-developer proxy and it correlates well in practice.
