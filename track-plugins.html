What this track teaches

🧠
The premise
Four small words run the whole Claude Code ecosystem — Skills, Plugins, MCP servers, Subagents. They sound similar, they get used interchangeably online, and most people pick the wrong one for the job. This track gives you a clean mental model, a curated short-list of what to install first, and a five-pattern guide for building your own when nothing off-the-shelf fits.

The 4-term vocabulary

Memorize this table once, then never confuse the four again. Each one has a specific job; you pick the one that matches the job, not whichever name sounds coolest.

The 4 Terms — What Each One Actually Is
TermWhat it isWhere it livesHow it loads
Skill A folder with SKILL.md + frontmatter. Encodes procedural knowledge — how to handle a class of task. ~/.claude/skills/ or project .claude/skills/ 30–50 tokens of metadata sit in context until the frontmatter description matches the request.
Plugin The distribution format. One plugin can bundle multiple skills, MCP servers, agents, and hooks. claude-plugins-official marketplace plus community sources /plugin install name@source
MCP server External connectivity — live access to GitHub, Postgres, Notion, a browser, etc. ~/.mcp.json or project .mcp.json Separate process. Its tool schemas load into context when the server is registered.
Subagent An isolated context window with a custom system prompt and a scoped tools list. ~/.claude/agents/ Spawned via the Task tool. Can route to a cheaper model.
💡
One-sentence relationship
Plugins distribute Skills. Skills add expertise. MCP servers add access. Subagents add execution capacity without polluting the main context.

Real workflow example

"Research competitor pricing and update the Notion database with what I find." That one request quietly uses every term in the table:

Skill
deep-research — the methodology: parallel search, source triangulation, citation verification.
↓
MCP server
@notionhq/notion-mcp-server — live write access to your Notion workspace.
↓
Subagent
research-analyst — isolates the long research context so your main session stays clean.
↓
Plugin
Notion@claude-plugins-official — the one-line install that brings the MCP server and several Notion skills along together.

You don't have to name any of this when you ask. Claude picks up the skill from the description match, the MCP server is registered once and stays available, and the subagent only spawns when the task is long enough to need it. The vocabulary helps you build, not chat.

5 install patterns

Every install method you'll see in the wild reduces to one of these five. Read the SKILL.md before you run any of them — treat skills like VS Code extensions, not magic.

01 Official marketplace (canonical)

/plugin install Notion@claude-plugins-official
Use when: the plugin exists in Anthropic's official marketplace. Lowest review burden.

02 Git clone (community skills)

git clone https://github.com/user/repo ~/.claude/skills/name
Use when: it's a community skill not yet in any marketplace. Open the repo. Read SKILL.md first.

03 openskills (cross-agent)

npx openskills install anthropics/skills
Use when: you want the same skill to work in Claude Code, Cursor, and Codex.

04 CCPM (third-party marketplace)

npx @daymade/ccpm install <skill-name>
Use when: browsing the 232+ skill CCPM directory for something specific.

05 ccpi (largest registry)

npx ccpi install <plugin>
Use when: you've already vetted a plugin from the 425-plugin / 2810-skill ccpi index. Variable quality — review carefully.

MCP server registration is separate from any of the above — it edits ~/.mcp.json:

{ "mcpServers": { "github": { "command": "npx", "args": ["-y", "@github/mcp-server"] } } }

Security baseline

🛡
Read this before installing anything

Skills run with your shell permissions. MCP servers run as long-lived processes with whatever credentials you hand them. Cato Networks 2025 research confirmed that malicious skills can ship through these channels disguised as productivity tools.

  • Open SKILL.md on GitHub before cloning. Read it. Look for "unrestricted," "--dangerously," "sudo," or "automatically execute."
  • Inspect any scripts/ directory. Never run a script you haven't read.
  • 53% of MCP servers in 2026 use static API keys with no rotation. Treat MCP credentials like database passwords.
  • Three CVEs were filed against Anthropic's own Git MCP server in January 2026 (path traversal + file deletion). Official does not mean immune — only better-monitored.
  • When uncertain, run the security-reviewer skill against the candidate — it exists for exactly this.

What to install first

Three starters that pay back immediately. Skip the rest until you have a real need.

1. The Notion plugin

Live read/write to your second brain. Pairs with notion-knowledge-capture and notion-spec-to-implementation for a complete workflow.
/plugin install Notion@claude-plugins-official

2. The Filesystem MCP

Path-restricted local file access for Claude Code — safer than running everything through Bash. Most-installed MCP server of 2026.
npx -y @modelcontextprotocol/server-filesystem ~/workspace

3. The superpowers plugin

Brainstorming, plan writing, TDD, verification — the most-installed Claude skills framework. Forces structure before output.
/plugin install superpowers@claude-plugins-official
